# Как получить изменения из PR или временной ветки

Эти шаги помогут скачать и просмотреть изменения, которые ещё не попали в ветку `main`.

## 1. Обновите ссылки на удалённый репозиторий
```bash
git fetch origin --prune
```
Это подтянет актуальный список веток и pull request'ов.

## 2. Проверьте, есть ли готовая ветка в origin
```bash
git branch -r | grep work
```
Если команда показала `origin/work`, можно создать локальную копию и перейти в неё:
```bash
git checkout -b work origin/work
```

## 3. Если ветки нет, скачайте pull request напрямую
Когда ветка автора ещё не опубликована в origin, но доступен pull request, его можно выкачать по номеру PR. Замените `<PR_NUMBER>` на нужный номер:
```bash
git fetch origin pull/<PR_NUMBER>/head:work
```
После этого переключитесь в новую ветку:
```bash
git checkout work
```

## 4. Проверьте полученные изменения
```bash
git status
```
Убедитесь, что вы находитесь в ветке `work`, и изучите список коммитов и файлов.

## 5. Обновление локальной ветки при новых правках
Если автор добавил новые коммиты в тот же pull request, выполните:
```bash
git checkout work
git fetch origin pull/<PR_NUMBER>/head:work --force
```
Команда перезапишет локальную ветку обновлённым состоянием PR.

Следуя этим шагам, вы сможете просматривать предложенные изменения до их слияния с `main`.
